How can we reduce data distortion and improve the efficiency of geospatial data transfer in early stage design workflows?

Reducing data distortion in multi-source geospatial workflows: A web based platform for efficient and accurate site analysis.

To develop a web based mapping platform that enables efficient and reliable site analysis through the integration of multiple geospatial datasets with a focus on clarity, usability, and practical site insight.

Site analysis in architecture and civil engineering often requires data from diverse sources such as OpenStreetMap, local planning portals, environmental datasets and elevation models. This data is typically manually collected, converted and imported into tools like AutoCAD or Revit, which is a time consuming process prone to data distortion, misalignment, and a loss of meta data.

This project aims to design a web based platform that streamlines multi source geospatial data integration, maintains semantic and geometric fidelity, and produces interoperable output formats useable in common design software. This approach will reduce time, errors and cost while improving the quality of the early design process.

The tool will not only visualise spatial data but also offer interpretative and evaluative functionality, enabling designers, planners, or researchers to assess how landscape features, built form, and proposed changes impact visual perception and character.

To create and evaluate a web based mapping platform that visualises spatial and environmental datasets and exports them in formats directly consumable by common design software.

Developing an Interactive Geospatial Platform for Designers

A web-based tool to visualise spatial data (e.g. land use, environmental conditions, infrastructure) and support design workflows in the built environment sector, with integration to industry design software.

Goals

  • Bridge the gap between raw geospatial data and design decision-making.
  • Support portable, site-responsive tools for designers (e.g. urban/landscape/architecture).
  • Explore how AI or visualisation techniques can enhance usability and insight.

Features Under Consideration

  • OSM-based interactive map (core foundation)
  • Overlay environmental data (contours, heights, land use)
  • AI-assisted 3D building form generation (for shadow/view analysis)
  • Visualise popular pedestrian routes
  • Show local materials / businesses (useful for construction teams)
  • Comment/draw layer — users annotate directly on map (site visit notes)
  • Hand-drawing tool (iPad/tablet friendly)
  • Y-axis mapping or section visualisation tool (aid section generation)

Objectives:

  • Automate collection of site data from open geospatial sources (e.g OSM, Planning APIs, terrain/elevation)
  • Normalise and validate geometries and data to ensure format fidelity
  • Provide visual in browser previews for designers/planners
  • enable export to industry standard formats (GeoJSON, DXF) with minimal loss/distortion
  • evaluate the solutions efficiency and accuracy compared to traditional workflows

Deliverables

  1. Functional web based prototype
  1. Data conversion engine with export capabilities
  1. Technical report documenting system architecture, testing and evaluation
  1. Research context, implementation and results

Selecting Appropriate Datasets

The project will include datasets that describe both physical form and perceptual qualities of a site. These include Digital Elevation Models (DEMs) for terrain understanding, tree and vegetation data for evaluating landscape texture and screening, land cover and land use zoning to define character types, heritage and listed buildings as cultural receptors, and transport infrastructure to map visual clutter and accessibility. Integration of planning constraint zones, viewpoints, and even photographic references could further enrich the platform’s contextual awareness.

Visualisation and Analytical Features

The platform's visualisation tools should be chosen not for technical complexity alone, but for their relevance to design and assessment tasks. Suggested features include:

  • Interactive elevation profiles that respond to user-drawn paths, aiding in understanding terrain impact.
  • Land character overlays using colour-coded zones based on existing classifications or derived land use.
  • Layer blending and toggles that allow scenario testing (e.g., existing vs proposed vegetation buffers).
  • Photographic viewpoints linked to map markers, connecting spatial data with ground-level perception.
  • Potentially, simplified viewshed simulations from user-defined locations could offer a novel interaction that approximates visual impact in a browser environment.

The value of these tools lies not only in their visual expressiveness but in their ability to support alternative design scenarios and inform planning.

The web mapping tool will be evaluated based on six main criteria: (1) web mapping functionality and feature coverage, (2) usability and interface, (3) use limits and restrictions, (4) performance and scalability, (5) quality and accuracy of geospatial data and (6) stability.
