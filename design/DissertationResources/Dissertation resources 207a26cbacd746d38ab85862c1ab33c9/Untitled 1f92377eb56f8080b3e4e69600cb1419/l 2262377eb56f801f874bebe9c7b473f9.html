Emerging web technologies such as HTML5, JavaScript, and WebGL have significantly advanced web mapping applications, transforming static maps into dynamic, interactive visualizations that allow users to zoom, pan, switch layers, and interact with timeline animations (Köbben and Kraak, 2020). The Open Geospatial Consortium (OGC) has been instrumental in developing standards like Web Map Services (WMS) and Web Feature Services (WFS) that enable interoperability across web mapping platforms (Neumann, 2012).

Geospatial data refers to information tied to geographic locations on Earth, often including coordinates (latitude and longitude), attributes, and sometimes temporal elements, making it spatio-temporal data when time is involved (Stock and Guesgen, 2016; Fanaee-T, 2012). Examples include weather station readings across time or traffic flow data.

Web maps are widely used in applications like weather forecasting, traffic navigation, logistics, and visualizing major events such as the COVID-19 pandemic or geopolitical conflicts. A variety of web mapping tools have simplified integration for developers, but choosing the right tool remains a challenge.

The term geovisualization encompasses a broad range of meanings—ranging from maps themselves to techniques and academic disciplines focused on visualizing geographic information interactively. As defined by Çöltekin et al. (2018), geovisualization is the interactive process of visualizing geographic data—spatial, temporal, or attribute-based—to support analysis, hypothesis generation, and decision-making through maps, graphs, and multimedia. It aims to facilitate knowledge construction and decision support by combining geospatial data with human interpretation (MacEachren et al., 2004; Laurini, 2017).

Web mapping refers to creating and delivering maps online, while web GIS extends this by enabling manipulation, analysis, and visualization of geospatial data on the web (Neumann, 2012; Veenendaal et al., 2017). The line between web maps and web GIS is increasingly blurred, as web maps now often include analytical features.

Geospatial data formats fall mainly into two categories:

  • Raster data represent geographic features as pixel arrays, commonly used for aerial or satellite imagery, in formats like PNG, TIFF, GIF, and JPEG (Williams, 2019).
  • Vector data use geometric shapes—points, lines, polygons—to model spatial features precisely. Popular formats include Esri Shapefile, KML, GeoJSON, and SVG (Diamond, 2019). Vector formats tailored for web and mobile include KML/KMZ, GeoJSON, and Vector Tiles.

The growth in computational power and data storage has driven an explosion in geospatial data availability from governments, universities, private companies, and individuals, often freely downloadable (Diamond, 2019).

Web maps can be categorized as:

  • Static maps: View-only, rarely updated, often scanned images of paper maps.
  • Dynamic maps: Generated on demand from live data sources via servers.
  • Distributed maps: Combine data from multiple sources using standards like WMS.
  • Animated maps: Show temporal changes (e.g., weather or traffic) via sequences of images.
  • Interactive maps: Enable user interaction—navigation, layer control, querying, and linking using event handling and scripting.

Additional specialized web maps include:

  • Real-time maps: Provide near-instant updates from sensors (e.g., live traffic or weather).
  • Personalized maps: Allow users to filter data and apply custom styles using standards like Styled Layer Description (SLD).
  • Open, reusable maps: Offer APIs for integration (e.g., Google Maps API), though often proprietary.
  • Analytic maps: Support spatial analysis, sometimes with server-side processing and client-side visualization.
  • Collaborative maps: Enable multiple users to co-create maps online, exemplified by projects like OpenStreetMap and WikiMapia.
