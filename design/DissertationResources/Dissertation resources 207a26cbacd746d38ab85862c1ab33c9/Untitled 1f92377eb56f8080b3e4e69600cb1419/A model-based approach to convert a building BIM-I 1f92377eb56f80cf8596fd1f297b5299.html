  • CityGML: A format used to model 3D cities (e.g., buildings, roads, terrain). It represents both geometry (shape) and semantics (meaning, like "this is a roof").Ideal for large-scale city modeling.
  • GML 3.2.1: A general XML-based language used by CityGML to define shapes and spatial properties (like surfaces, curves).
  • IFC (Industry Foundation Classes) is used for buildings, and it assumes a flat Earth. This can introduce distortions over long distances. If you assume the Earth is flat (like IFC does), you'll get errors in coordinates over long distances. A scale factor is used to correct this when using map projections (to account for the Earth's curve).
  • Triple Graph Theory: A concept in data integration used to link two different models (like IFC and CityGML) using a third graph that maps elements between them. Helps automate data conversion while preserving relationships.
  • FZKViewer: A visualization tool developed by the Forschungszentrum Karlsruhe (FZK) in Germany. It's used to view IFC and CityGML files for inspection and validation of 3D models.
  • LOD (Level of Detail):Refers to how detailed a 3D model is. LOD400–LOD500: Highly detailed BIM models. LOD4 in CityGML: Includes interior building structures (e.g., rooms). Conservative transfer means preserving as much detail and accuracy as possible when converting between formats.
  • FME (Feature Manipulation Engine): A software that allows for data conversion, transformation, and integration between formats like IFC and CityGML. It's a drag-and-drop tool for geospatial workflows.
  • Shell Objects in IFC: Shells are closed surfaces that form the shape of a 3D object in IFC. Think of them as "skins" wrapping around solids to represent walls, floors, etc.
  • Cubic spline: A smooth curve defined by control points; keeps the correct curvature.
  • Linear interpolation: Connects points with straight lines; curvature = infinity (not realistic for curves).
  • Loss of Accuracy (Biljecki et al. 2018): Using models with different Levels of Detail or inaccurate data can lead to wrong results in simulations.

  • Ray tracing: A technique from computer graphics to determine what is visible from a viewpoint. Used to identify which surfaces are on the outside of a 3D object.
  • Even/Odd Rule (EOR): Counts how many times a ray crosses surfaces to decide if a point is inside or outside.
  • Z-buffer test: Speeds up processing by focusing only on the visible surfaces, instead of checking all.
  • Multi-threading: A programming method that allows a computer to do many calculations at the same time. Very useful for heavy 3D computations like ray tracing.
  • Semantic + Geometry Integration (Arroyo Ohori et al. 2018):Together, they assess how accurately a model has been converted from IFC to CityGML.
    ifcOpenShell    : A tool for handling IFC files and extracting their semantic data (e.g., identifying a wall or a roof).
    CGAL (Computational Geometry Algorithms Library): A C++ library used to analyze and process geometry(e.g., distances, shapes).
  • ML algorithms can label parts of a 3D model (e.g., "this triangle is part of a roof").
  • Heuristic rules: Simple rules based on geometry (e.g., orientation of surfaces).
  • Kappa coefficient: A metric to measure accuracy of the labeling (above 0.8 = good, 0.99 = excellent).
  • Think of voxels like 3D pixels. They break complex geometry into small cubes to make it easier to convert from IFC to CityGML. Helpful for automating geometric analysis.
  • Before converting a model, users often: Visually inspect the model. Check random elements to understand the structure (like checking if it uses IFC4). Use tools like Solibri for deeper checks (rules, standards, errors).
  • CityGML 3.0 and AbstractOccupiedSpace: A new concept in CityGML 3.0 to help represent occupied space at different levels of detail (LOD1–3). Helps make IFC-to-CityGML conversion more accurate by reducing semantic gaps (differences in meaning).
