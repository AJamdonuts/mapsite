How can we reduce data distortion and improve the efficiency of geospatial data transfer in early stage design workflows?

Reducing data distortion in multi-source geospatial workflows: A web based platform for efficient and accurate site analysis.

To develop a web based mapping platform that enables efficient and reliable site analysis through the integration of multiple geospatial datasets with a focus on clarity, usability, and practical site insight.

Site analysis in architecture and civil engineering often requires data from diverse sources such as OpenStreetMap, local planning portals, environmental datasets and elevation models. This data is typically manually collected, converted and imported into tools like AutoCAD or Revit, which is a time consuming process prone to data distortion, misalignment, and a loss of meta data.

This project aims to design a web based platform that streamlines multi source geospatial data integration, maintains semantic and geometric fidelity, and produces interoperable output formats useable in common design software. This approach will reduce time, errors and cost while improving the quality of the early design process.

Existing tools:

  • Require software e.g Autodesk
  • Not built for cross discipline
  • Not built for early stage collaboration
  • dont address data fidelity in transfer/ conversion

Each platform has a different system for setting up the map.- Different tagging systems are used during data collection. e.g OSM label a street as “highway:””residential”, while a council database might use “type”:”local road”.

Overlays from different datasets can misalign.- Different Coordinate Reference Systems (CRS), Differing scales/ resolutions, Inconsistent digitisation.

Existing platforms like Google Maps, Arc GIS Online, and Leaflet-based apps usually display data as is. User must then manually sort data so that it all matches, to be able to integrate it further.

They do not warn you of overlays, gaps or misalignments. Data used may then be guessed/not accurate or will require manual tweaking. QGIS does let a manual fix but requires knowledge of GIS.

No change simulation built on clean data

Most existing platforms either visualise data without allowing interaction or editing, or they allow editing without showing potential consequences. They also have their own data not caring about others so often grabbing data or using different platforms becomes uncertain when double checking as its unsure of which sites has distortion.
